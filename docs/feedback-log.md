# Журнал пользовательского фидбека

Каждая запись: кто (роль, не имя), что сказал/зарепортил, что сделано в ответ.
Сырые слова пользователя важнее пересказа — цитируем. Ссылка на issue обязательна,
на fix-коммит/PR — как только есть.

|---|---|---|---|---|
| — | — | — | — | — |

## Как сюда попадает фидбек

1. Пользователь заводит issue по шаблону («🐞 Что-то не работает» / «💬 Идея») —
   или пишет письмом/в мессенджер, тогда issue заводим за него с пометкой `via-email`.
2. Каждое issue с ярлыком `user-feedback` появляется строкой в этой таблице.
3. Реакция — одно из: `fixed <commit/PR>` · `planned <milestone>` · `declined + почему`.
   Молча закрывать нельзя.
