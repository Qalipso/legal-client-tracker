# Письмо: как пользоваться Legal Client Tracker

> Черновик для отправки.

---

**Тема: Legal Client Tracker — доступ и как начать**

Привет!

Собрал доступ к Legal Client Tracker — мини-CRM для ведения клиентов и дел.

## Ссылки

- **Приложение (прод):** https://legal-client-tracker.vercel.app
- **Telegram-бот (уведомления):** https://t.me/legalclienttrackerbot
- **Репозиторий (код + документация):** https://github.com/Qalipso/legal-client-tracker

## Тестовый доступ

```
Email:   test@qalipso.legal
Пароль:  testtest
```

Аккаунт создан через штатную панель Supabase (Authentication → Add user,
с автоподтверждением) — реальный рабочий логин, не заглушка. Данные внутри
чистые (демо-режим), можно смело нажимать что угодно.

## Как пользоваться — коротко

1. **Войти** по ссылке выше тестовыми кредами.
2. **Доска дел** — канбан по статусам (Новый / В работе / Ожидает клиента /
   Закрыт). Карточку можно перетащить в другую колонку — статус сменится
   сам. Есть вид «Таблица» рядом с «Доска».
3. **Добавить клиента** — кнопка в шапке; обязательны только имя и телефон,
   остальное можно дозаполнить потом.
4. **Открыть карточку клиента** (клик по имени) — там всё дело целиком:
   сводка (тип дела, стадия, контрольный срок), стороны, задачи, контрольные
   сроки, риски/открытые вопросы, документы, полная история изменений.
5. **Настройки** (⚙ в шапке):
   - профиль — имя, компания, фото, роль;
   - уведомления — включить Telegram и выбрать, о чём уведомлять;
   - получатели — сюда добавляется твой Telegram chat ID;
   - импорт/экспорт клиентов в CSV;
   - история отправленных уведомлений.
6. **Telegram-уведомления**: Настройки → Получатели → «📎 Подключить Telegram» —
   откроется Telegram, нажми Start, бот подтвердит подключение сам (chat ID
   вписывать вручную не нужно). Нажми «Отправить тест» — придёт тестовое
   сообщение. После этого бот сам пришлёт уведомление при добавлении нового
   клиента, просрочке или смене статуса дела.
7. **История и аналитика** (📊 в шапке) — пока заглушка, отчёты будут позже.

## Что важно знать

- Данные каждого аккаунта видны только ему — это разные, изолированные
  друг от друга рабочие пространства.
- Роль по умолчанию — «Юрист» (полный доступ к своим данным). Есть роль
  «Ассистент» с урезанными правами (не может удалять клиентов и менять
  настройки уведомлений) — назначается вручную, не через интерфейс.
- Документы дела пока хранятся как имя файла без содержимого (прототип);
  фото профиля — уже полноценная загрузка.

Если что-то непонятно или неудобно — пиши, поправим.
