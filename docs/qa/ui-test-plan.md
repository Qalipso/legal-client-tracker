# UI Test Plan — Legal Client Tracker

Методология: полноэкранный headless-браузер, два разрешения — desktop
**1920×1080** и mobile **375×812** (iPhone-класс). Каждый экран: скриншот +
`preview_inspect` для точных значений стилей/размеров там, где скриншота
недостаточно. Пройдено 2026-07-09 по живому приложению (Supabase-режим,
тестовый аккаунт).

## Найденные проблемы

| # | Экран | Серьёзность | Проблема | Статус |
|---|---|---|---|---|
| 1 | Header (mobile) | **High** | `flex items-start justify-between` не переносит чип пользователя и группу кнопок на отдельные строки на узких экранах → кнопки сжимаются, текст «+ Добавить клиента» ломается на 3 строки | ✅ Исправлено |
| 2 | Settings → Уведомления (mobile) | Medium | Заголовок секции и статус-бейдж в одной `flex justify-between` строке; бейдж с длинным текстом переносится некрасиво поверх заголовка | ✅ Исправлено |
| 3 | Notify function / История уведомлений | Medium | В причине пропуска события показывается сырой код `event_type` (например, `событие «status.changed» выключено...`) вместо человекочитаемого названия | ✅ Исправлено |
| 4 | Settings → Получатели | Low | Кнопка «Send test notification» — единственная английская строка в русском интерфейсе | ✅ Исправлено |
| 5 | Client Drawer (desktop, широкие мониторы) | Low | Фиксированная ширина 448px на экране 1920px — длинная вертикальная прокрутка при богатой карточке дела | ✅ Исправлено (адаптивная ширина на `lg`/`xl`) |
| 6 | Settings — подзаголовок | Low | «Профиль, уведомления и получатели» не упоминает разделы «Данные» и «История», которые уже есть на странице | ✅ Исправлено |

## Проверенные и отклонённые гипотезы (не баги)

- **Board/Table «не центрированы» на 1920px** — при инспекции `mx-auto max-w-7xl`
  реально даёт равные отступы (344px слева/справа); визуальное ощущение
  перекоса — из-за левого выравнивания текста внутри колонки, не баг контейнера.
- **Client Drawer «обрезан» на мобильном** — при инспекции `width: 375px,
  right: 0` — драйвер реально на весь экран; полоска в скриншоте —
  скроллбар headless-браузера при 2x DPI рендере скриншота.

## Что не покрыто в этом проходе (дальше)

- Table view на мобильном (узкие горизонтальные таблицы — вероятно нужен
  horizontal scroll wrapper, не проверено)
- Tablet-разрешение (768px) — промежуточные breakpoints не проверялись
  отдельно, только mobile/desktop крайности
- Keyboard-навигация и screen-reader проход (a11y-аудит — отдельная задача)
- Real device testing (только headless Chrome; touch-специфичные проблемы,
  такие как HTML5 drag-and-drop не работающий на touch, не тестировались
  интерактивно — известное ограничение, см. README Next steps)
